Help & FAQs

1. What is SpamShield?

SpamShield is a machine learning-powered web app that classifies SMS messages as either "Spam" or "Ham" (not spam) using natural language processing and ensemble models.

2. How accurate is the spam detection?

The model achieves over 95% accuracy on the test set from the dataset it was trained on. However, it may not perform as well on messages with slang, code-mixed text, or other languages.

3. Do you store the messages I submit?

No, SpamShield does not store or share any messages. All processing is done temporarily in-memory during your session for classification purposes only.

4. Can I use this for business or production?

SpamShield is designed for educational and demo purposes only. For commercial or production use, we recommend further model validation, optimization, and security audits.

5. What format should the bulk upload file follow?

Please upload a .csv file with a column named message containing SMS texts only. You can download a sample here.

Please upload a .txt file containing SMS texts only. You can download a sample here.

6. What if the classifier is wrong?

While we strive for high accuracy, no model is perfect. You can always rephrase the message or report feedback through our Contact page.

7. Which technologies were used?

SpamShield uses Python, Flask, Scikit-learn, TF-IDF, and a Voting Classifier (SVM + Naive Bayes + Extra Trees), styled using HTML, CSS, and JavaScript.

8. How do I classify a single message?

Go to the Classify page, enter your SMS text, and click "Classify" to see the result instantly.

9. How do I classify multiple messages at once?

Go to the Bulk Upload page and upload a CSV file with your messages. The model will classify all of them for you.

10. Is login required?

Login is optional. You can use all features without logging in, but user accounts may let you save your history in future updates.

11. Who can I contact for support?

Use the Contact page to send us feedback or questions. We'll be happy to help!
